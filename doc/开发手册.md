## 2. 编程环境与依赖配置

本项目基于 B/S 架构 ，采用前后端分离及控制面/执行面解耦的设计。系统涉及前端（Vue）、后端（Java）和算法端（Python） 的混合编程，必须严格统一环境版本。



### 2.1 硬件与操作系统要求

- **开发环境：** Windows 10/11 或 macOS 13+
- **运行环境（服务器）：** Ubuntu 20.04 LTS 或 CentOS 7.9
- **内存要求：** 开发机建议 16GB 以上（因需同时运行 IDEA, PyCharm, WebStorm 及中间件）。

### 2.2 前端开发环境 (Vue.js)

- **Node.js:** `v18.17.0` (LTS 版本)
- **包管理器:** `pnpm 8.x` 或 `npm 9.x`
- **核心框架:** `Vue 3.3` + `Vite 4.4` + `Element Plus 2.3`
- **图表可视化库:** `ECharts 5.4` (用于绘制胜率与价值评估曲线)

### 2.3 后端业务控制端环境 (Java)

- **JDK 版本:** `OpenJDK 17`
- **构建工具:** `Maven 3.8.6+`
- **核心框架:** `Spring Boot 3.1.5` + `MyBatis-Plus 3.5.3`
- **开发工具 IDE:** IntelliJ IDEA 2023.2+

### 2.4 算法执行端环境 (Python)

- **解释器:** `Python 3.9.18` (必须使用 Miniconda 隔离虚拟环境)
- **核心框架:** `FastAPI 0.103` (用于暴露 HTTP 接口接收 Java 调度指令)
- **AI 与仿真依赖:** `PyTorch 2.1.0` + `PettingZoo 1.24` (多智能体强化学习环境)

2.5 中间件与数据库配置



- 
  **关系型数据库:** `MySQL 8.0.33`
- **缓存与消息代理:** `Redis 7.0+` (用于 Python 训练日志的异步队列，Java 消费后通过 WebSocket 推送给前端)

## 3. 工程目录结构规范

为了规范团队协作，降低代码冲突，各端的工程目录结构必须严格遵循以下规范。

### 3.1 Java 后端目录结构 (Spring Boot)

后端采用标准的分层架构（Controller-Service-Mapper），并独立出调度模块。

Plaintext

```
game-ai-backend/
├── src/main/java/com/nuaa/gameai/
│   ├── controller/      # RESTful API 接口层 (处理前端请求)
│   ├── service/         # 业务逻辑层 (任务编排、场景配置处理)
│   ├── mapper/          # 数据库访问层 (MyBatis-Plus 接口)
│   ├── entity/          # 实体类层 (对应 MySQL 表结构)
│   ├── websocket/       # WebSocket 服务端 (用于向前端推送实时训练指标)
│   ├── pythonclient/    # Python 服务调用客户端 (封装 HttpUtil)
│   └── config/          # 系统配置类 (Redis, CORS, WebSocket 等配置)
└── src/main/resources/
    ├── application.yml  # 核心配置文件 (数据库连接、Redis 地址等)
    └── mapper/          # MyBatis XML 映射文件

```

### 3.2 Python 算法端目录结构 (FastAPI + RL)

Plaintext

```
game-ai-engine/
├── app/
│   ├── api/             # FastAPI 路由定义 (如 /start_train, /stop_train)
│   ├── rl_algorithms/   # 强化学习算法库 (PPO, DQN 等算法的实现)
│   ├── environments/    # 无人机红蓝对抗仿真环境定义 (基于 PettingZoo)
│   ├── core/            # 核心业务逻辑 (训练循环控制、Redis 状态回写)
│   └── main.py          # FastAPI 服务启动入口
└── requirements.txt     # Python 依赖清单

```

> **【扩充页数建议】：** 将前端 Vue 的完整目录结构（`src/views`, `src/components`, `src/store` 等）也补充进来。同时，可以将后端的 `pom.xml` 文件内容和 Python 的 `requirements.txt` 文件内容完整地粘贴到文档中，这不仅规范，而且能有效扩充篇幅。

4. 数据库设计与初始化配置



本项目数据库命名为 `game_ai_db`，字符集统一采用 `utf8mb4`。

### 4.1 核心数据表 DDL 脚本

系统设计中包含场景配置、算法配置及训练任务等核心实体。以下为初始化 SQL 脚本：

SQL

```
-- 1. 场景配置表 (记录战场网格、无人机数量等)
CREATE TABLE `scene_config` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `scene_name` VARCHAR(100) NOT NULL COMMENT '场景名称',
  `map_width` INT NOT NULL COMMENT '地图宽度',
  `map_height` INT NOT NULL COMMENT '地图高度',
  `red_drone_count` INT NOT NULL COMMENT '红方无人机数量',
  `blue_drone_count` INT NOT NULL COMMENT '蓝方无人机数量',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='博弈对抗场景配置表';

-- 2. 训练任务表 (记录智能体训练任务的生命周期)
CREATE TABLE `training_task` (
  `task_id` VARCHAR(64) PRIMARY KEY COMMENT '分布式任务流水号 UUID',
  `scene_id` BIGINT NOT NULL COMMENT '关联场景ID',
  `algo_name` VARCHAR(50) NOT NULL COMMENT '使用的算法名称 (如 MAPPO)',
  `hyper_parameters` JSON NOT NULL COMMENT '算法超参数(学习率,奖励折扣等)',
  `status` TINYINT NOT NULL DEFAULT 0 COMMENT '0-待启动, 1-训练中, 2-已完成, 3-异常',
  `current_epoch` INT DEFAULT 0 COMMENT '当前训练轮数',
  `start_time` DATETIME COMMENT '任务启动时间',
  `end_time` DATETIME COMMENT '任务结束时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='智能体训练任务调度表';

```

> **【扩充页数建议】：** 继续补充“算法配置表 (`algorithm_config`)”、“用户表 (`sys_user`)” 和“训练指标日志表 (`training_log`)”的建表 SQL。针对每个表，画一个 Markdown 表格详细解释每个字段的作用。

## 5. 跨语言系统通信规范

本系统最核心的难点在于 Java 调度平台与 Python 训练引擎的交互。统一采用 HTTP + Redis 的异步通信模式。

### 5.1 Java 下发训练指令 (HTTP POST)

前端在页面点击“开始训练”后，Java 后端接收请求，落库生成 `task_id`，随后调用 Python 端的 API。

- **接口地址：** `http://{python_host}:8000/api/train/start`
- **调用方：** Java Spring Boot
- **服务方：** Python FastAPI
- **请求体 (JSON):**

JSON

```
{
  "task_id": "T-20260321-0001",
  "scene_config": {
    "red_count": 5,
    "blue_count": 5
  },
  "algo_config": {
    "learning_rate": 0.001,
    "max_epochs": 1000
  }
}

```

### 5.2 Python 回传训练指标 (Redis Pub/Sub)

由于强化学习训练耗时极长，Python 进程在每一个 Epoch 结束时，需将当前的态势价值评估和胜率写入 Redis。

- **Redis 队列/频道名：** `channel:train:metrics:{task_id}`
- **数据载荷 (JSON):**

JSON

```
{
  "task_id": "T-20260321-0001",
  "epoch": 150,
  "red_win_rate": 0.65,
  "loss_value": 0.102,
  "timestamp": "2026-03-21T10:30:00Z"
}

```

- **说明：** Java 端需配置 RedisListener 监听此频道，一旦收到数据，立刻通过 WebSocket 转发给前端 Vue 的 ECharts 进行图表渲染。

## 6. 系统启动与本地联调指南

在本地进行前后端及算法端联调时，请严格按照以下顺序启动服务：

1. **启动基础设施：**
  - 启动 MySQL 服务，执行第 4 章的初始化 SQL。
  - 启动 Redis 服务 (默认端口 6379)。
2. **启动 Python 算法引擎：**
  - 进入 `game-ai-engine` 目录，激活 conda 环境。
  - 运行指令：`uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload`
3. **启动 Java 业务控制端：**
  - 在 IDEA 中运行 `GameAiApplication.java` 主类 (默认端口 8080)。
4. **启动 Vue 前端界面：**
  - 进入前端目录，运行 `pnpm dev`，打开浏览器访问 `http://localhost:5173`。
