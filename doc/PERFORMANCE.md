# 非功能需求自测说明（摘要）

| 指标（需求文档 4.1） | 建议做法 |
|---------------------|----------|
| 常规操作 ≤ 1.5s | 浏览器 DevTools → Network，对 `POST /api/scenes` 等保存接口记录 TTFB+下载时间，截图存档。 |
| 实时监控 ≤ 2s | 训练页观察 WebSocket 消息间隔；或在同一机器上打印 Redis 发布时间与前端接收时间差。 |
| 并发训练 | 连续提交两个「立即启动」任务，验证第二个为 **排队** 状态，并在第一个完成后自动开始。 |

说明：实际数值与机器与 GPU/CPU 负载相关，本文件用于课程设计文档引用，请在答辩前本地实测并填写具体数值。
